
Playwright:
--------------
Playwright it is an API for end to end to modern web application testing


By default Playwright executes all test cases in parallel and 
executes on (chromium,firefox,webkit).

By default playwright executes all test in headless mode
Headless:
-------------

------------------------------

2.To run specifi test case
npx playwright test tests/testcasename

--------------------------

Types
=======

2.Soft Assertion

How to execute Javascript code in Playwright
======================================
What evaluate() does
It executes the function you pass in the page’s context, not in Node.js.
This means it can access browser objects like:
document,window,localStorage

Why use evaluate()?
--------------------------
To read or modify the DOM
To run custom JavaScript inside the webpage
To get values not directly exposed by Playwright API

evaluate() is more flexible when accessing any page-level JavaScript.

//scrolldown
await page.evaluate(()=>window.scrollTo(0,document.body.scrollHeight));
//scrollup
await page.evaluate(()=>window.scrollTo(document.body.scrollHeight,0));
//scroll for Element
await page.locator("").scrollIntoViewIfNeeded();

Alert handling
=======================
Alerts are browser based and for any application wrong selectOptionsalert will trigger

There are 3 types of Alert
==========================
1.Simple alert(alert)
2.Confirmation alert(confirm)
3.prompt alert(prompt)

-Playwright can interact with the web page dialogs such as alert, confirm, prompt 
-By default, dialogs are auto-dismissed by Playwright
-But to perform specific action you can manually trigger it using page.on listener


for page.on event handling alert 4 methods are available
--------------------------------
1.OK-->To simulate action ok = accept()
2.Cancel-->To simiulate action on Cancel =dismiss()
3.To get alert text: message()
4.To check type of alter= type()
5.for Prompt alert = accept("prompt")

//Note
=======
- trigger alert listener always before opening that alert

===================

	page.waitForEvent('popup'),
	page.locator().click() which tigger that event

special note
===================
1.don't use await in promise.all()
 Because any of the promise fullfill,it will return immediately.
2.dont change sequence also.

first wait for the event, tren trigger the event by clicking on it. 

3. dont use await page. locator().click() & await page.waitfor Event() 
- It will be running in Sequential mode.

after click() promise fullfill and it returns immediately and it won't capture
the waitForEvent() which look for that event(newtab)so always 
use without awiat and so it should be in same sequence as well.

Assertions
===================
To validate certain state of application we use Assertions

Playwright support two types of assertion
1.Hard assertion
--------------------
If assertion fail playwright will stop execution on that line
and after 5sec you will see assertion error

2.Soft assertion
---------------------
If assertion fail playwright will never stop execution on that line,
it will continue till and show error on that line


Assertions in playwright we can implement using expect()

We have auto wait for assertions which is 5sec.
